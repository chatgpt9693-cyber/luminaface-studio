## 🔧 Как настроить зоны

### Шаг 1: Включить debug
```bash
npm run dev
```
В браузере нажми кнопку "🔧 Debug" слева вверху.

### Шаг 2: Визуально оценить
Увидишь розовые wireframe сферы на модели. Если зона не попадает:

### Шаг 3: Скорректировать позиции
Открой `src/components/FaceModel.tsx`, найди `ZONE_HITS`:

```typescript
// Пример: сдвинуть лоб выше
{ id: 'forehead', pos: [0, 0.40, 0.12], scale: [0.22, 0.10, 0.08] }
//                           ↑ было 0.35, стало 0.40
```

**Координаты:**
- `pos[0]` — X (лево/право): `-` влево, `+` вправо
- `pos[1]` — Y (верх/низ): `-` вниз, `+` вверх
- `pos[2]` — Z (глубина): `-` назад, `+` вперед

**Масштаб:**
- `scale[0]` — ширина
- `scale[1]` — высота
- `scale[2]` — глубина

### Шаг 4: Hot reload
Vite автоматически перезагрузит — сразу увидишь изменения.

### Шаг 5: Выключить debug
Нажми "🔧 Debug ON" чтобы вернуться в нормальный режим.

## 📐 Система координат

```
       Y (вверх)
       ↑
       |
       |
       +----→ X (вправо)
      /
     /
    Z (вперёд к камере)
```

Модель центрирована в `[0, 0, 0]`.

## 💡 Советы

1. **Начни с Y-координаты** — самая важная для попадания
2. **Используй маленькие шаги** — 0.05-0.1 за раз
3. **Проверяй на разных углах** — покрути модель мышкой
4. **Масштаб важен** — слишком большие зоны перекрываются

## 🎨 Визуальная настройка

В debug режиме:
- **Розовый wireframe** — зона видна
- **Прозрачность 0.4** — не мешает видеть модель
- **Клик работает** — можно тестировать интерактивность
